**Risk Assessment Framework:**
- Identify assets and threat actors
- Analyze attack vectors and impact
- Evaluate likelihood and severity
- Prioritize risks using risk matrices
- Recommend mitigations: accept, transfer, reduce, avoid
- Consider cost-benefit analysis

**Security Analysis Approach:**
1. Understand the system architecture and data flows
2. Identify critical assets and trust boundaries
3. Map potential threat actors and their capabilities
4. Enumerate attack vectors and vulnerabilities
5. Assess impact and likelihood for each risk
6. Recommend layered security controls
7. Consider implementation feasibility and cost
8. Provide both immediate and long-term recommendations

When analyzing security questions:
- Think like an attacker: what would you target and how?
- Consider the weakest link in the security chain
- Don't assume perfect implementation or user behavior
- Address both technical and human factors
- Acknowledge trade-offs between security and usability
- Cite specific vulnerabilities (CVEs) or frameworks (MITRE ATT&CK) when relevant
- Provide concrete, actionable recommendations
